new 연산자와 생성자 함수

사용하는이유

객체 리터럴을({...}) 사용하면 객체를 쉽게만들수있습니다 하지만 유사항 객체를 만들어야할때가있습니다 복수사용자,메뉴내 아이템을 객체로 편하려할때 그런경우죠

여기서 new연산자와 생성자 함수를 사용하여 유사한 객체를 여러개를 생성할수있습니다



생성자 함수

생성자 함수는 손쉽게 사용자 객체를 만들 수 있습니다. 그리고 읽기쉽게 만들수도잇죠

생성자 함수는 일반적인 함수와 기술적인 차이는 없지만 두 관계를따릅니다

  • 함수 이름의 첫 글자는 대문자로 시작합니다.
  • 반드시 'new' 연산자를 붙여 실행합니다.

주의할점은 여기서 모든 함수는 생성자 함수가 될 수 있습니다.


생성자와 return문

생성자 함수엔 보통 return이 없습니다 반환해야 할건 this에 저장되고 자동으로 변환되기 때문입니다.

만약 return이 있다면 어떻게 될까요?

  • 객체를 return 한다면 this 대신 객체가 반환됩니다.
  • 원시형을 return 한다면 return문이 무시됩니다.

요약

  • 생성자 함수(짧게 줄여서 생성자)는 일반 함수입니다. 다만, 일반 함수와 구분하기 위해 함수 이름 첫 글자를 대문자로 씁니다.
  • 생성자 함수는 반드시 new 연산자와 함께 호출해야 합니다. new와 함께 호출하면 내부에서 this가 암시적으로 만들어지고, 마지막엔 this가 반환됩니다.

그리고 잘 이해가 안되서 이 영상 참고했습니다
